项目经历

MuseFlow AI - AI 驱动的智能音乐播放平台

Vue 3 TypeScript Vite Pinia Vue Router Express SSE Sass HTML5 Audio DashScope / OpenAI Compatible API

项目描述：

基于 Vue 3 + TypeScript 构建的智能音乐播放平台，整合搜索、歌单、MV、评论、播放队列、收藏/本地音乐、最近播放同步与 AI 自然语言控播能力；通过 Node 中间层聚合音乐数据接口与大模型接口，形成从音乐发现、播放控制到用户音乐资产沉淀的完整闭环。

项目亮点：

  • 流式响应体验优化：基于 SSE 实现 AI 回复逐段输出与中断控制，用户无需等待完整结果返回即可看到响应内容，减少 AI 交互过程中的等待空白感，提升对话实时性。
  • 路由懒加载与页面拆包：对首页、搜索、歌单、MV、设置、迷你播放器等业务页面进行路由级懒加载，减少首屏一次性加载资源；生产构建中多数业务页 JS 控制在约 4KB - 17KB，主布局 gzip 后约 19.5KB。
  • AI 点歌准确率优化：基于 OpenAI 兼容接口接入 Qwen 模型，通过 Prompt 约束结构化输出，将自然语言意图收敛为“推荐 / 搜索 / 切歌 / 播放”等动作；覆盖 30+ 条常见点歌指令手测，点歌意图识别与动作命中率约 90%+。
  • 搜索与内容消费体验优化：搜索页支持歌曲/歌单/MV 分类、分页、URL 状态同步和过期请求丢弃，并可一键将当前页可播放歌曲转为播放队列；同时补齐评论、MV 播放、歌手/专辑跳转等链路，提升从发现歌曲到播放消费的转化效率。
  • 播放状态持久化节流：对进度条拖动、音量调整等高频操作进行节流保存，降低频繁写入 localStorage 带来的性能损耗，同时保留刷新后恢复播放进度、音量和播放模式的能力。

MindCare Assistant - AI 智能咨询与服务管理平台

Vue 3 TypeScript Vite Pinia Vue Router Element Plus Axios ECharts wangEditor Node.js

项目描述：

基于 Vue 3 + TypeScript 构建的 AI 智能咨询与后台管理平台，前台提供问题咨询、反馈提交和建议展示能力，后台提供咨询记录管理、内容维护和数据统计看板；前端使用 Element Plus 搭建管理端页面，结合 Pinia、Axios、ECharts 和 wangEditor 完成状态管理、接口请求、数据可视化与富文本内容编辑。

项目亮点：

  • 管理端页面组件化：基于 Element Plus 搭建表格、分页、弹窗、表单校验等后台高频交互，统一处理加载状态、空状态和错误提示，提升咨询记录与反馈数据的管理效率。
  • 接口请求与状态管理：使用 Axios 封装咨询、反馈、内容维护等接口请求，配合 Pinia 管理用户信息、咨询列表和后台筛选条件，减少跨页面数据传递和重复请求逻辑。
  • 数据统计可视化：基于 ECharts 实现咨询量、问题类型、处理状态等统计图表，并结合筛选条件动态刷新数据，帮助后台更直观地观察服务压力和用户关注点。
  • 富文本内容维护：接入 wangEditor 实现知识库、公告或回复模板编辑，配合表单回显、校验和提交流程，支持后台沉淀可复用的问题解决方案。
