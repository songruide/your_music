数据库第七章课后作业 E-R 图

按教材常见画法绘制：矩形表示实体，椭圆表示属性，菱形表示联系；主码属性加下划线，多值属性用双椭圆，连线旁标明联系类型。

实体 属性 联系 下划线：主码 1 / n / m：联系类型

1. 销售部、职工、产品

转换后的关系模式

销售部(部门名称, 地区, 电话)

职工(职工号, 姓名, 性别, 部门名称)

产品(产品编号, 品名, 单价)

销售(部门名称, 产品编号, 数量)

外键：职工.部门名称 → 销售部.部门名称；销售.部门名称 → 销售部.部门名称；销售.产品编号 → 产品.产品编号。

2（1）顾客有多个地址和多个电话号码

转换后的关系模式

顾客(身份证号, 姓名, 卡号, 等级, 积分)

顾客地址(身份证号, 省, 市, 区, 街道)

顾客电话(身份证号, 区号, 号码)

外键：顾客地址.身份证号 → 顾客.身份证号；顾客电话.身份证号 → 顾客.身份证号。卡号也可以作为候选码。

2（2）顾客有多个地址，每个地址有多个电话号码

转换后的关系模式

顾客(身份证号, 姓名, 卡号, 等级, 积分)

顾客地址(身份证号, 省, 市, 区, 街道)

地址电话(身份证号, 省, 市, 区, 街道, 区号, 号码)

外键：顾客地址.身份证号 → 顾客.身份证号；地址电话的身份证号、省、市、区、街道 → 顾客地址对应主码。

3. 部门、员工、实习生、项目

转换后的关系模式

部门(部门编号, 部门名称, 办公地点, 经理员工编号)

员工(员工编号, 姓名, 级别, 部门编号)

实习生(实习编号, 姓名, 年龄, 部门编号, 项目编号)

项目(项目编号, 项目名称, 开始日期, 结束日期, 负责人员工编号)

员工参与项目(员工编号, 项目编号, 工作时间比)

外键：员工.部门编号 → 部门.部门编号；部门.经理员工编号 → 员工.员工编号；实习生.部门编号 → 部门.部门编号；实习生.项目编号 → 项目.项目编号；项目.负责人员工编号 → 员工.员工编号；员工参与项目中的员工编号、项目编号分别引用员工和项目。
