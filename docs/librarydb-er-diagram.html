librarydb 图书借阅管理系统 E-R 图

根据 MySQL 数据库 librarydb 的实际表结构，并结合实验报告模板中“图书借阅管理系统”的描述绘制。图中采用《数据库系统概论》常见 E-R 图样式：实体为矩形，属性为椭圆，联系为菱形，主码属性加下划线。

实体 属性 联系 下划线：主码 1 / n / m：联系类型

一、实体属性 E-R 图分析

1. 图书类别实体属性 E-R 图

图书类别实体包含图书类别编号、类别名称、类别描述等属性，其中图书类别编号为主码。

2. 图书实体属性 E-R 图

图书实体包含图书编号、书名、作者、出版社、出版日期、价格、库存数量等属性，其中图书编号为主码。

3. 读者类别实体属性 E-R 图

读者类别实体包含读者类别编号、类别名称、最大借阅数量、最大借阅天数等属性，其中读者类别编号为主码。

4. 读者实体属性 E-R 图

读者实体包含读者编号、姓名、性别、电话、注册日期等属性，其中读者编号为主码。

二、联系分析

在图书管理系统中，一个读者类别可以包含多个读者，一个图书类别可以包含多个图书，所以这两组联系都是一对多。一个读者可以借阅多本图书，一本图书也可以在不同时间被不同读者借阅，所以读者与图书之间是多对多联系，借阅日期、应归还日期、实际归还日期等属于“借阅”联系的属性。

四、优化初步 E-R 图后的基本 E-R 图

优化时去掉不必要的重复外码属性，只保留实体本身属性和联系。图书类别编号、读者类别编号在逻辑模型中作为外码进入图书表和读者表，在 E-R 图中用“包含”联系表达即可。

关系说明

图书类别 与 图书：1:n，一个图书类别可以包含多本图书，一本图书只属于一个图书类别。

读者类别 与 读者：1:n，一个读者类别可以包含多个读者，一个读者只属于一个读者类别。

读者 与 图书：m:n，通过“借阅”联系关联；借阅编号、借阅日期、应归还日期、实际归还日期是该联系的属性。

由 E-R 图转换得到的关系模式

图书类别(BC_No, BC_Name, BC_Desc)

图书(B_No, B_Name, Author, Publisher, PubDate, Price, Stock, BC_No)

读者类别(RC_No, RC_Name, MaxBorrowQty, MaxBorrowDays)

读者(R_No, R_Name, Gender, Phone, RegDate, RC_No)

借阅(BorrowNo, R_No, B_No, BorrowDate, DueDate, ReturnDate)

主码和外码

图书类别主码：BC_No；图书主码：B_No；读者类别主码：RC_No；读者主码：R_No；借阅主码：BorrowNo。

图书外码：BC_No → 图书类别.BC_No。

读者外码：RC_No → 读者类别.RC_No。

借阅外码：R_No → 读者.R_No；B_No → 图书.B_No。

数据库中的 vw_unreturnedbooks 和 vw_unreturnedmorethan1 是视图，用来查询未归还图书信息，不作为基本实体画入 E-R 图。
